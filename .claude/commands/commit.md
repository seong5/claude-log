# Claude 명령어: Commit

이모지와 컨벤셔널 커밋 메시지로 잘 포맷된 커밋을 생성합니다.

## 프로세스

1. `git status`로 스테이지된 파일 확인
2. 스테이지된 파일이 있으면 해당 파일만 커밋, 없으면 전체 변경사항 분석
3. `git diff --staged` (또는 `git diff`)로 변경 내용 분석
4. 여러 논리적 관심사가 섞여 있으면 분할 커밋 제안
5. 아래 포맷으로 커밋 생성

## 커밋 포맷

`<이모지> <타입>: <설명>`

**타입과 이모지:**

| 이모지 | 타입 | 설명 |
|--------|------|------|
| ✨ | feat | 새로운 기능 |
| 🐛 | fix | 버그 수정 |
| 📝 | docs | 문서화 |
| 💄 | style | 포맷팅 |
| ♻️ | refactor | 코드 리팩토링 |
| ⚡️ | perf | 성능 개선 |
| ✅ | test | 테스트 |
| 🔧 | chore | 빌드/도구 |
| 🚀 | ci | CI/CD |
| 🗑️ | chore | 불필요한 코드 제거 |
| 💥 | feat | 브레이킹 체인지 |
| 🔒️ | fix | 보안 수정 |
| 🌐 | feat | 국제화/현지화 |
| ♿️ | feat | 접근성 개선 |

**규칙:**

- 명령형 어조 사용 ("추가" not "추가됨")
- 첫 줄 72자 미만
- 원자적 커밋 (하나의 논리적 변경사항)
- **커밋에 Claude 서명 절대 추가하지 않음**

## 분할 기준

다음에 해당하면 분할 커밋을 제안합니다:
- 서로 다른 관심사가 혼재할 때
- feat + fix처럼 타입이 혼합될 때
- 특정 파일 패턴이 독립적인 변경사항일 때
- 변경 규모가 매우 클 때

## 실행

```bash
git status
git diff --staged  # 스테이지된 파일이 있을 경우
git diff           # 스테이지된 파일이 없을 경우
```

분석 후 적절한 커밋 메시지를 생성하고 커밋을 실행합니다.
