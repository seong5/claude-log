# Claude Log 개발 로드맵

> **최종 업데이트**: 2026-04-23  
> **버전**: 1.0.0  
> **상태**: Active  
> **현재 브랜치**: feature/main-logic

## 요약

Claude Log는 Claude Code 토큰 사용량을 실시간 추적하고 GitHub 잔디 형태로 시각화하는 macOS 메뉴바 앱이다. Electron + React 19 + Tailwind CSS v4 + Zustand v5 기반으로 개발 중이며, `~/.claude/projects/**/*.jsonl` 파일을 로컬에서 파싱하여 별도 서버 없이 동작한다. 현재 MVP 핵심 기능 대부분이 구현되어 있으며, 비용 계산 기능과 패키징 검증이 남아 있다.

## 성공 지표

- [ ] 일간 활성 사용: 앱 팝업 1회 이상 오픈
- [ ] 데이터 파싱 정확도: 실제 토큰 수 대비 +-5% 이내
- [ ] 앱 시작 시간: 1.5초 미만
- [ ] 메모리 사용량: 상주 중 80MB 미만
- [ ] 크래시 없는 사용: 7일 연속 무크래시

## 아키텍처 개요

```
메인 프로세스 (src/main/)
  - index.ts: Tray + BrowserWindow(frameless popup) 관리
  - jsonl-parser.ts: JSONL 파싱 엔진 (증분 오프셋 기반)
  - log-service.ts: fs.watch 기반 실시간 감지 + 데이터 집계

프리로드 (src/preload/)
  - contextBridge로 claudeLog API 노출 (getDays, onUpdate)

렌더러 (src/renderer/src/)
  - App.tsx: 메인 레이아웃 (헤더/히트맵/통계/7일차트)
  - TokenHeatmap.tsx: GitHub 잔디형 히트맵 (1년/이번달 토글)
  - StatsPanel.tsx: 요약 통계 카드 + 입출력 비율 + 모델별 사용량
  - useLogStore.ts: Zustand 스토어 (IPC 기반 데이터 동기화)
```

## 개발 단계

### Phase 1: MVP 완성 (예상 1.5주)
**목표**: PRD v0.1 기능을 모두 구현하고 .dmg로 배포 가능한 상태 달성  
**상태**: 진행 중

#### 마일스톤

- [x] **M1.1** Tray 기반 메뉴바 앱 구조 - Tray 아이콘, frameless 팝업, Dock 숨김 `완료`
- [x] **M1.2** JSONL 파싱 엔진 - 증분 파싱, fs.watch 실시간 감지, 디렉토리 감시 `완료`
- [x] **M1.3** 히트맵 대시보드 UI - GitHub 잔디형 히트맵, 기간 토글, 툴팁 `완료`
- [x] **M1.4** 요약 통계 패널 - 스트릭, 총 토큰, 세션, 모델별 비율, 입출력 비율, 7일 차트 `완료`
- [ ] **M1.5** cache 토큰 분리 - cache_read/cache_creation을 input과 분리하여 파싱 및 UI 표시 `예상 2일`
- [ ] **M1.6** 예상 비용 계산 - 모델별 단가 적용, 비용 추정 카드 UI `예상 2일`
- [ ] **M1.7** Claude Code 활성 상태 감지 - 사용 중 아이콘 색상 변경 `예상 1일`
- [ ] **M1.8** 오늘/이번주/이번달 요약 카드 보강 - 헤더 또는 통계 영역에 3단위 요약 `예상 1일`
- [ ] **M1.9** macOS .dmg 패키징 검증 - electron-builder로 빌드 테스트 및 코드 사이닝 `예상 2일`

#### 산출물
- cache 토큰 분리가 적용된 파싱 엔진
- 모델별 비용 계산 기능이 포함된 통계 패널
- 검증된 macOS .dmg 설치 파일

#### 완료 기준
- [ ] cache_read, cache_creation 토큰이 UI에서 별도로 표시된다
- [ ] 모델별 단가에 따른 예상 비용이 달러 단위로 표시된다
- [ ] Claude Code 프로세스 활성 시 Tray 아이콘이 시각적으로 구분된다
- [ ] .dmg 파일로 설치 후 정상 동작한다
- [ ] 1년치 데이터 파싱이 2초 미만에 완료된다
